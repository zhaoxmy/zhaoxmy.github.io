A - Not Too Hard

原题链接:A-Not Too Hard

难度：入门

算法：无

题解：

要求解n个数里面所有小于等于x的数的和,逐个求解即可，不加赘述

AC code

#include <iostream>
using namespace std;
int main(){
    int n,x;
    cin>>n>>x;
    long long ans=0;
    int a=0;
    for(int i=0;i<n;i++){
        cin>>a;
        if(a<=x)
            ans+=a;
    }
    cout<<ans;
}

B - 11/11

原题链接:B - 11/11

难度：入门

算法：无

题解：

数据范围最多有100个月，每个月最多100天，暴力枚举每一天即可。
比较麻烦的是如何判断一个日期中每一个整数都相同。我的方案是利用tostring函数转换为字符串，然后判断。

AC code

#include <iostream>
#include <vector>
using namespace std;
bool ck(string b){
    char te=b[0];
    for(int i=1;i<b.length();i++)
        if(b[i]!=te)
            return 0;
    return 1;
}
int main(){
    int n;
    cin>>n;
    vector<int> days(n);
    for(int i=0;i<n;i++)
        cin>>days[i];
    int ans=0;
    for(int i=1;i<=n;i++){
        for(int j=1;j<=days[i-1];j++)
            if(ck(to_string(i)+ to_string(j)))
                ans++;
    }
    cout<<ans;
}

C - Consecutive

原题链接:C - Consecutive

难度：中级–

算法：前缀和

题解：

前缀和的模板问题，考虑声明一个数组v，表示字符串s相应位置的贡献。
具体的说，若sp==sp+1,
则vp=1，
否则vp0。
问题转换为求解数组v在区间[l,r)上的和，典型前缀和问题

11-15日更新，想了想把这道题从初级改到了中级–。中级–表示题目本身难度只有初级，但初级水平的选手很难解的题。前缀和是非常简单的算法，但是并不常见于初级题目中，属于有点偏门的类型。前缀和，差分，字典树，回文树，kmp等等。这些算法都非常简单，但要是一个选手都能会，都能写的话，那这个选手应该处于中级水准，而非初级水准。

AC code

#include <iostream>
#include <vector>
using namespace std;
template<typename T>
void reverseDifference(vector<T> &a){
    for(int i=1;i<a.size();i++)
        a[i]+=a[i-1];
}
int main(){
    int n,q;
    cin>>n>>q;
    string a;
    cin>>a;
    vector<int> v(n+1);
    for(int i=0;i<n-1;i++)
        if(a[i]==a[i+1])
            v[i+1]++;
    reverseDifference(v);
    int l,r;
    for(int i=0;i<q;i++){
        cin>>l>>r;
        cout<<v[r-1]-v[l-1]<<endl;
    }
}

D - Take ABC

原题链接:D - Take ABC

难度：初级

算法：顺序表

题解一

题目要求删除所有连续的ABC，一个可行的思路是：每遇到一次c，就检查他的前两位是不是a,b，如果是，则将其删除，复杂度瓶颈在于数组删除所需o(n)时间，n次就需要n2。因为复杂度瓶颈在于删除，所以很显然可以考虑链表。
在实现上，由于直接链表中删除节点会导致对应的迭代器失效。所以实现为对于每一个节点，检查他的前三个是不是c,b,a,若是，则删除前三个。
由于list不支持访问前三个，只支持访问前一个，所以重载了list迭代器的加法和减法运算符，以便代码实现。

AC code

#include <list>
#include <iostream>
using namespace std;
template<typename T>
_List_iterator<T> operator+(_List_iterator<T> a, int x){
    for(int i=0;i<x;i++)
        a++;
    return a;
}
template<typename T>
_List_iterator<T> operator-(_List_iterator<T> a, int x){
    for(int i=0;i<x;i++)
        a--;
    return a;
}
int main(){
    string a;
    cin>>a;
    list<char> li;
    for(char i : a)
        li.push_back(i);
    list<char>::iterator cd;
    cd=li.begin();
    cd=cd+3;
    for(;cd!=li.end();cd++){
        if(*(cd-1)=='C' && *(cd-2)=='B' && *(cd-3)=='A')
            li.erase(cd-3,cd);
    }
    if(*(cd-1)=='C' && *(cd-2)=='B' && *(cd-3)=='A')
        li.erase(cd-3,cd);
    for(auto i:li)
        cout<<i;
}

题解二

赛后拜读大佬代码，得到了一种新的解题思路。
首先我们知道，每次删除末尾的C字母下标是单调递增的，那我们完全可以维护一个vector，如果队列尾部三个分别是’A’’B’’C’则弹出队列末尾三个，然后不断将s加入vector的末位。最终得到的vector即为答案。

AC code

#include <iostream>
#include <vector>

using namespace std;
int main(){
    string a;
    cin>>a;
    vector<char> v;
    for(auto i:a){
        v.push_back(i);
        if(v.size()>=3 && *(v.end()-1)=='C' && *(v.end()-2)=='B' && *(v.end()-3)=='A')
            for(int w=0;w<3;w++)
                v.pop_back();
    }
    for(auto i:v)
        cout<<i;
}

D-总结

虽然D只是一道初级的顺序表问题，但在却能在一定程度上考验选手对不同顺序表的理解能力。题解二的写法仅仅只有16行，而且仅需要题解一三分之一的运行时间
作为最广为人知的stl容器，vector对于任何基于线性表的算法都有着优秀的适配能力，如果仅仅把vector视作动态数组，你将会失去c++的一大宝藏

E - Modulo MST

原题链接:E - Modulo MST

难度：初级+

算法：搜索，并查集

题解

前言

这道题不论是赛时完成还是赛后整理都花了我大量的时间，大搜索和大模拟确实是比赛时遇到的比较头疼的两种题，尤其是像acm这种题量大，时间紧的比赛。

思路

闲话少叙，看到这道题第一眼相信大家第一反应就是最小生成树问题。但可以的是，这道题的答案是模意义上的，也就是非线性的，或者说不符合交换律的。他并不能跑最小生成树算法。外加数据范围只有最多8个点，28条边，答案已经很明显了，这个题的正解就是剪枝暴搜。

实现

那么接下来要做的事情就是——构建一颗简洁而优雅的搜索树，即思考如何才能枚举每一种合法的状态。
plan A:
暴力枚举每一条边是否存在，每一条边有存在和不存在两种可能，然后判合法，保存权值的最小值。复杂度为228=268,435,456约为3e8，说实话这个复杂度有点不美丽，这意味着你必须要有极低的常数才能ac，但是判合法的常数很难降低。所以考虑剪枝
plan B:
很显然可以注意到，最小生成树有且只可能有n-1条边，剪掉所有边数大于n-1的节点。也可以视作从m个边里选出n-1条边，他们两个是等价的。复杂度降低为C287。可以接受

判断是否合法

一般来说，判合法的算法应该是前置知识，只需要照搬最小生成树算法中判合法的算法就好了。但鉴于这道题只有初级+，而最小生成树的算法一般来说是中级，所以顺道提一嘴。利用并查集把一条被选的边的两端的点所在的集合合并，如果仅用被选出的边就将所有点合为一个集合，那么合法，否则不合法。

一些优化

并查集判合法是根据边来进行的，所以可以一边选边一边判合法。能够有常数级的优化（由于本人写的很烂，可能是负优化）。AC code使用了这种写法

AC code

#include <iostream>
#include <vector>
typedef long long ll;
using namespace std;
ll n,m,k,ans=1e18,temp=0;
class e1{
public:
    ll x,y,w;
    e1(ll a,ll b,ll c){
        x=a;
        y=b;
        w=c;
    }
};
vector<e1> edge;
template<typename DA>
class unionFindSets{
public:
    vector<DA> data;
    vector<int> fa;
    int inquire(int x){
        if(fa[x]==x)
            return x;
        fa[x]=inquire(fa[x]);
        return fa[x];
    }
    void merge(int a,int b){
        fa[inquire(a)]=fa[inquire(b)];
    }
    unionFindSets(int size){
        data.resize(size);
        fa.resize(size);
        for(int i=0;i<size;i++)
            fa[i]=i;
    }
};
void qans(int l,int num,unionFindSets<int> a){
    if(num==0){

        ans=min(ans,temp%k);
        return;
    }
    for(int i=l;i<edge.size();i++){
        if(a.inquire(edge[i].x)!=a.inquire(edge[i].y)){
            unionFindSets<int> s(a);
            s.merge(edge[i].x,edge[i].y);
            temp+=edge[i].w;
            qans(i+1,num-1,s);
            temp-=edge[i].w;
        }
    }
}
int main(){
    cin>>n>>m>>k;
    ll x,y,w;
    for(int i=0;i<m;i++){
        cin>>x>>y>>w;
        edge.emplace_back(x-1,y-1,w);
    }
    unionFindSets<int> s(n);
    qans(0,n-1,s);
    cout<<ans<<endl;
}
