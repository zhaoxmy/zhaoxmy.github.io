A. Game with Integers

原题链接:A. Game with Integers

难度：入门

算法：博弈论

题解：

人类智慧题，不难发现，不管vanya怎么修改这个数，vova都能下一次让他变回去。所以得到结论：“如果vanya能赢，则他只需要一步就能赢”。问题转化为一个数在一步之后能不能修改为3的倍数。很显然的，除非这个数本身是3的倍数，不然必然能够在一次操作后变成3的倍数。

AC code

#include <iostream>
using namespace std;
int main(){
    int t0;
    cin>>t0;
    int a;
    for(int i=0;i<t0;i++){
        cin>>a;
        if(a%3)
            cout<<"First"<<endl;
        else
            cout<<"Second"<<endl;
    }
}

B. 250 Thousand Tons of TNT

原题链接:B. 250 Thousand Tons of TNT

难度：初级-

算法：数论

题解：

非常具有俄罗斯风味的题，看到这道题我的第一反应是排序后贪心，都快码出来了，结果突然反应过来题目的数据具有全序性，遂考虑dp，然后发现这题有不了一点最有子结构，后陷入迷茫。考虑到暴力的复杂度是O(nk)k是n的因数个数，突然反应过来这道题可能是诈骗题，遂交了一发暴力，然后AC。事后整理发现，n的因数个数最多是2sqrt(n)个。警惕电信诈骗。解法思路看注释吧。
#include

#include
using namespace std;
int main(){
long long t0,n,ans,temp=0,maxt,mint;
vector v;
cin>>t0;
for(int t=0;t<t0;t++){
ans=0;
cin>>n;
v.clear();
v.resize(n);
for(auto &i:v)//输入
cin>>i;
for(int i=1;i<n;i++){
if(!(n%i)){//如果能整除
maxt=0;//最大的一辆车
mint=1e17;//最小的一辆车
for(int j=0;j<n/i;j++){
temp=0;
for(int i1=0;i1<i;i1++)//计算每一辆车
temp+=v[j*i+i1];
maxt=max(temp,maxt);//保存最大最小
mint=min(temp,mint);
}
ans=max(ans,maxt-mint);//更新答案
}
}
cout<<ans<<endl;
}
}

C. Yarik and Array

原题链接:C. Yarik and Array

难度：简单

算法：动规

题解：

这题就真的是一眼动规了，很明显，想在以下标i位置的数结尾的子数组si,想在他后边再接一个数，要求数组和最大，si必然要取最大。所以该题是无后效性的，拥有最有子结构的。不管怎么说，反正他是可以动规的。
注：坑点一，子数组必须连续
动规维护一个dp数组，dp[i]表示以i结尾的最大子数组的和。如果v[i]和v[i-1]的奇偶性不同，那么dp[i]可以是dp[i-1]+v[i]，也可以是单独的v[i]，否则只能是单独的v[i]，动态转移方程如下：
dp[i]=(abs(v[i-1]%2)!=abs(v[i]%2))?max(v[i],dp[i-1]+v[i]):v[i];

AC code

#include <iostream>
#include <vector>

using namespace std;
int main(){
    int t0;
    cin>>t0;
    int n;
    for(int t=0;t<t0;t++) {
        cin >> n;
        int maxAns = -1e7;
        vector<int> v(n + 1,-1e7);
        vector<int> dp(n + 1,-1e7);
        for(int i=1;i<=n;i++)
            cin>>v[i];
        for(int i=1;i<=n;i++){
            dp[i]=(abs(v[i-1]%2)!=abs(v[i]%2))?max(v[i],dp[i-1]+v[i]):v[i];
            maxAns=max(maxAns,dp[i]);
        }
        cout<<maxAns<<endl;
    }
}

D. Yarik and Musical Notes

原题链接:D. Yarik and Musical Notes

难度：中级-

算法：数论，离散化

题解：

题目大意是给定一个数组a,要求有多少对i，j,（i小于j），使得（2ai)2aj==(2aj)2ai.
这个式子一看就非常的炸裂,
我们对他进行一些化简，得到ai-log2ai=aj-log2aj
对y=x-log2x求导，得y’=1-1/(xln2);
分析得，函数在（0，log2e]上单调递减，在（log2e,+无穷）上单调递增。
因此得出推论，若满足题目关系式，要么ai=aj,要么两者分布在log2e的左右两边。
又因为，i，j均为正整数，所以，要么ai=aj，要么ai,aj分别为1和2.分析到这一步，题目就迎刃而解了。
只需要简单离散化求解数据里每一种数有几个。需要注意的是，multimap和multiset的count函数复杂度在极端情况下极为炸裂，会达到O(n)，请勿使用。

AC code

给出两种不同的实现方式
方式一 采用数组排序进行离散化（易于理解实现）：

#include <iostream>
#include <vector>
#include <algorithm>
typedef long long ll;
using namespace std;
int main(){
    int t0;
    cin>>t0;
    int n;
    for(int t=0;t<t0;t++) {
        cin >> n;
        vector<int> v(n);
        for(auto &i:v)
            cin>>i;
        ll num1=0,num2=0,num3=0;
        ll len=0,num=0;
        sort(v.begin(),v.end());
        for(auto temp:v){
            if(temp==1)
                num1++;
            if(temp==2)
                num2++;
            if(temp==num)
                len++;
            else{
                num3+=len*(len-1)/2;
                len=1;
                num=temp;
            }
        }
        num3+=len*(len-1)/2;
        cout<<num1*num2+num3<<endl;
    }
}

方式二:巧用map进行离散化（推荐）：

#include <iostream>
#include <map>
typedef long long ll;
using namespace std;
int main(){
    int t0,n;
    cin>>t0;
    for(int t=0;t<t0;t++) {
        cin >> n;
        ll a,ans=0;
        map<ll,ll> m;
        for(int i=0;i<n;i++){
            cin>>a;
            if(m.count(a))
                m.find(a)->second++;
            else
                m.insert(pair<int,int>(a,1));
        }
        for(auto i:m)
            ans+=i.second*(i.second-1)/2;
        if(m.count(1) && m.count(2))
            ans+=m.find(1)->second*m.find(2)->second;
        cout<<ans<<endl;
    }
}
