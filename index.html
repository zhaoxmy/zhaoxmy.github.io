并查集/路径压缩并查集

发表于 2023-11-16 更新于 2023-11-17 分类于 acmCodeLibrary ， contioners

概述

相信大家都知道数学里的集合这一概念（并不是STL里的set），集合这一概念在数学中应用甚广。算法作为一门研究如何计算的技术，自然而然会需要进行集合的运算。然而遗憾的是，目前并没有找到能在较低时间里进行所有集合运算的数据结构。因此对于不同的的集合运算，我们需要选用不同的数据结构来实现。
这就引出了我们今天的主题——并查集，用于在O(log*n)时间里实现：
1.将两个集合合并
2.查询某个元素所属的集合
注：log*n在n高达265536时，仅仅为6，实际应用中可以认为复杂度等同于O(1)
很显然，支持以上操作，就可以求解具备可传递性的问题，最典型的就是联通性问题

树状结构储存集合关系

兴许你在学习并查集的时候不一定学过树这种数据结构，但是无妨，并查集所用的树状结构是被简化过的。实际上他已经去掉了树状结构的大部分形体，只保留了一些神韵。但个人之见要彻底理解并查集，还得从树状结构讲起。若不在乎原理，可以直接跳到路径压缩并查集的实现。

利用树/图的父子关系来表示集合的父子关系

在集合问题上，我们通常会说，某个集合是另一个集合的子集。某个元素属于某个集合，某个集合有哪些元素。现在，我们用一个图来表示这种关系

既然有了图，就会需要一些算法来求解图上问题。但是图上算法的复杂度并不乐观，也一直没有好的算法来进行集合的运算，好在对于特殊规则下的集合，我们可以跑特殊的算法。
并查集所需的特殊规则是：每一个元素仅属于一个集合，大概就是这个样子

具体实现

通过上图不难看出来，并查集其实就是对于每个元素，维护一个指向他父亲（又称为集合号）的指针。这样我们可以很轻松的实现O(1)查找，问题的关键在于如何并。

方式一

很显然有一种最暴力的思路，对于每一次合并，就把其中一个集合中所有元素的指针指向另一个集合号。但是这样合并的复杂度就到了O（n）

方式二

另外一个思路是，当两个集合合并的时候，把一个集合的记号号指向另一个集合的集合号，如图：


但是这样查询的复杂度就等同于树高，最坏可以到O(n)

神奇的优化：重构并查集

现在让我们来思考一个问题，假设我现在有一个已经用方式二进行了很多次合并的并查集，他现在应该有很高的树高。我们现在要查询他，那我们有没有算法可以把方式二建出来的并查集给他摊平，修改成方式一的并查集呢？
很明显，我们只需要遍历所有节点，把所有节点都指向根节点就可以了。这个时候我们发现，合并总用时O(n)，摊平O(n),每次查询O(1)。我们实质上实现了平均单次合并，查询均为O(1)时间。
虽说这是一个先并再查的特例，但是我们是怎么做到的？时间是从哪里省出来的？不是特例的情况下能用同样的方法省时间吗？
我们可以从不同的角度来分析这一优化
从方法一的角度来说，每一个节点都要经历逐级被搬到根节点的子节点的过程，但是我们使用了某种意义上的懒操作。我们不再逐级往上搬，而是一步到位直接搬到根节点，将多次操作合并为一次。
从方法二的角度来说，我们在查询一个子节点的时候，发现他的根节点等于他父亲的根节点，因此我们可以运用记忆化的思想，在查询一个节点的根节点时复用他父亲的答案。
综合分析来看，我们只需要一次性处理尽可能多的合并，并且将合并的数据保存下来，以供复用，我们就能够节省大量的时间。

路径压缩并查集

要一次处理尽可能多的合并，那么最显然且最优方法就是运用贪心的思路。等到你查询，我再处理合并，而且只处理查询位置及其祖先们的合并。比方说对于如下并查集结构


假设我要查询节点9，那么11和12就可复用9的查询结果，降低复杂度
就像这个样子


数据范围越大，该方法获得的优化越多，最终被证明平均值为O(log*n)，也就是和O（1）相差无几。

路径压缩并查集的实现

通过以上分析，我们知道并查集的实现原理，我们只需要实现两个函数：
1.查询x节点，实际合并（递归实现）{
查询x节点的父节点；
将自己并到根节点底下，即让自己的父指针指向根节点
}
2.将x节点所在集合并入y节点所在集合{
查询x节点，得到x的根节点
查询y节点，得到y的根节点
让x的根节点成为y的根节点的孩子
}
通过之前讨论，这两个函数都有接近O(1)的复杂度

模板代码

模板代码采用数组模拟指针实现

#include <vector>
#include <iostream>

using namespace std;
template<typename DA>
class unionFindSets{
public:
    vector<DA> data;//存储节点数据
    vector<int> fa;//fa[x]表示x的父节点下标
    int inquire(int x){//查询函数，返回x的根节点下表
        if(fa[x]==x)//递归终点
            return x;
        fa[x]=inquire(fa[x]);//实际合并，也叫路径压缩
        return fa[x];
    }
    void merge(int a,int b){//合并函数
        fa[inquire(a)]=fa[inquire(b)];
    }
    unionFindSets(int size){//构造函数
        data.resize(size);
        fa.resize(size);
        for(int i=0;i<size;i++)//将每一个节点的父亲初始化为自己，方便编码，避免野指针
            fa[i]=i;
    }
};

B - 11/11

原题链接:B - 11/11

难度：入门

算法：无

题解：

数据范围最多有100个月，每个月最多100天，暴力枚举每一天即可。
比较麻烦的是如何判断一个日期中每一个整数都相同。我的方案是利用tostring函数转换为字符串，然后判断。

AC code

#include <iostream>
#include <vector>
using namespace std;
bool ck(string b){
    char te=b[0];
    for(int i=1;i<b.length();i++)
        if(b[i]!=te)
            return 0;
    return 1;
}
int main(){
    int n;
    cin>>n;
    vector<int> days(n);
    for(int i=0;i<n;i++)
        cin>>days[i];
    int ans=0;
    for(int i=1;i<=n;i++){
        for(int j=1;j<=days[i-1];j++)
            if(ck(to_string(i)+ to_string(j)))
                ans++;
    }
    cout<<ans;
}

D - Take ABC

原题链接:D - Take ABC

难度：初级

算法：顺序表

题解一

题目要求删除所有连续的ABC，一个可行的思路是：每遇到一次c，就检查他的前两位是不是a,b，如果是，则将其删除，复杂度瓶颈在于数组删除所需o(n)时间，n次就需要n2。因为复杂度瓶颈在于删除，所以很显然可以考虑链表。
在实现上，由于直接链表中删除节点会导致对应的迭代器失效。所以实现为对于每一个节点，检查他的前三个是不是c,b,a,若是，则删除前三个。
由于list不支持访问前三个，只支持访问前一个，所以重载了list迭代器的加法和减法运算符，以便代码实现。

AC code

#include <list>
#include <iostream>
using namespace std;
template<typename T>
_List_iterator<T> operator+(_List_iterator<T> a, int x){
    for(int i=0;i<x;i++)
        a++;
    return a;
}
template<typename T>
_List_iterator<T> operator-(_List_iterator<T> a, int x){
    for(int i=0;i<x;i++)
        a--;
    return a;
}
int main(){
    string a;
    cin>>a;
    list<char> li;
    for(char i : a)
        li.push_back(i);
    list<char>::iterator cd;
    cd=li.begin();
    cd=cd+3;
    for(;cd!=li.end();cd++){
        if(*(cd-1)=='C' && *(cd-2)=='B' && *(cd-3)=='A')
            li.erase(cd-3,cd);
    }
    if(*(cd-1)=='C' && *(cd-2)=='B' && *(cd-3)=='A')
        li.erase(cd-3,cd);
    for(auto i:li)
        cout<<i;
}

题解二

赛后拜读大佬代码，得到了一种新的解题思路。
首先我们知道，每次删除末尾的C字母下标是单调递增的，那我们完全可以维护一个vector，如果队列尾部三个分别是’A’’B’’C’则弹出队列末尾三个，然后不断将s加入vector的末位。最终得到的vector即为答案。

AC code

#include <iostream>
#include <vector>

using namespace std;
int main(){
    string a;
    cin>>a;
    vector<char> v;
    for(auto i:a){
        v.push_back(i);
        if(v.size()>=3 && *(v.end()-1)=='C' && *(v.end()-2)=='B' && *(v.end()-3)=='A')
            for(int w=0;w<3;w++)
                v.pop_back();
    }
    for(auto i:v)
        cout<<i;
}

E - Modulo MST

原题链接:E - Modulo MST

难度：初级+

算法：搜索，并查集

题解

前言

这道题不论是赛时完成还是赛后整理都花了我大量的时间，大搜索和大模拟确实是比赛时遇到的比较头疼的两种题，尤其是像acm这种题量大，时间紧的比赛。

思路

闲话少叙，看到这道题第一眼相信大家第一反应就是最小生成树问题。但可以的是，这道题的答案是模意义上的，也就是非线性的，或者说不符合交换律的。他并不能跑最小生成树算法。外加数据范围只有最多8个点，28条边，答案已经很明显了，这个题的正解就是剪枝暴搜。

实现

那么接下来要做的事情就是——构建一颗简洁而优雅的搜索树，即思考如何才能枚举每一种合法的状态。
plan A:
暴力枚举每一条边是否存在，每一条边有存在和不存在两种可能，然后判合法，保存权值的最小值。复杂度为228=268,435,456约为3e8，说实话这个复杂度有点不美丽，这意味着你必须要有极低的常数才能ac，但是判合法的常数很难降低。所以考虑剪枝
plan B:
很显然可以注意到，最小生成树有且只可能有n-1条边，剪掉所有边数大于n-1的节点。也可以视作从m个边里选出n-1条边，他们两个是等价的。复杂度降低为C287。可以接受

判断是否合法

一般来说，判合法的算法应该是前置知识，只需要照搬最小生成树算法中判合法的算法就好了。但鉴于这道题只有初级+，而最小生成树的算法一般来说是中级，所以顺道提一嘴。利用并查集把一条被选的边的两端的点所在的集合合并，如果仅用被选出的边就将所有点合为一个集合，那么合法，否则不合法。

一些优化

并查集判合法是根据边来进行的，所以可以一边选边一边判合法。能够有常数级的优化（由于本人写的很烂，可能是负优化）。AC code使用了这种写法

AC code

#include <iostream>
#include <vector>
typedef long long ll;
using namespace std;
ll n,m,k,ans=1e18,temp=0;
class e1{
public:
    ll x,y,w;
    e1(ll a,ll b,ll c){
        x=a;
        y=b;
        w=c;
    }
};
vector<e1> edge;
template<typename DA>
class unionFindSets{
public:
    vector<DA> data;
    vector<int> fa;
    int inquire(int x){
        if(fa[x]==x)
            return x;
        fa[x]=inquire(fa[x]);
        return fa[x];
    }
    void merge(int a,int b){
        fa[inquire(a)]=fa[inquire(b)];
    }
    unionFindSets(int size){
        data.resize(size);
        fa.resize(size);
        for(int i=0;i<size;i++)
            fa[i]=i;
    }
};
void qans(int l,int num,unionFindSets<int> a){
    if(num==0){

        ans=min(ans,temp%k);
        return;
    }
    for(int i=l;i<edge.size();i++){
        if(a.inquire(edge[i].x)!=a.inquire(edge[i].y)){
            unionFindSets<int> s(a);
            s.merge(edge[i].x,edge[i].y);
            temp+=edge[i].w;
            qans(i+1,num-1,s);
            temp-=edge[i].w;
        }
    }
}
int main(){
    cin>>n>>m>>k;
    ll x,y,w;
    for(int i=0;i<m;i++){
        cin>>x>>y>>w;
        edge.emplace_back(x-1,y-1,w);
    }
    unionFindSets<int> s(n);
    qans(0,n-1,s);
    cout<<ans<<endl;
}
